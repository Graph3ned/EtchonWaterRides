# Test Cases Table Description

## Overview
The test cases are presented in a structured table format containing 39 comprehensive test cases designed to validate the Boat Rental Management System (BRMS). The table uses a systematic organization with four main columns that provide complete test documentation.

## Table Structure

### Column Headers
The table contains the following four columns:

1. **No.** - Sequential numbering from 1 to 39
2. **Modules / Unit** - Functional module or system component being tested
3. **Test Case** - Descriptive name of the specific test scenario
4. **Cases** - Expected outcomes and validation criteria

## Test Case Distribution by Module

### Authentication (Test Cases 1-4)
- **Test Cases**: 4 tests
- **Focus**: User login validation, role-based access, security baseline, and session management
- **Coverage**: Admin login, staff login, authentication security, and session timeout

### Access Control (Test Cases 5-6)
- **Test Cases**: 2 tests
- **Focus**: Unauthorized access prevention and staff privilege restrictions
- **Coverage**: URL protection and role-based function access

### Staff Management (Test Case 7)
- **Test Cases**: 1 test
- **Focus**: Duplicate prevention in staff account creation
- **Coverage**: Data integrity in administrative functions

### Ride Rentals (Test Cases 8-20)
- **Test Cases**: 13 tests (largest module)
- **Focus**: Core business functionality including rental creation, editing, completion, and validation
- **Coverage**: Transaction management, real-time updates, automated calculations, life jacket tracking, billing, and data validation

### Reports & Sales (Test Cases 21-29)
- **Test Cases**: 9 tests
- **Focus**: Business intelligence and reporting capabilities
- **Coverage**: Financial reports, operational reports, date filtering, staff filtering, CSV exports, and growth calculations

### Ride Availability (Test Cases 30-35)
- **Test Cases**: 6 tests
- **Focus**: Real-time availability management and customer service
- **Coverage**: Availability viewing, filtering, real-time updates, and mobile responsiveness

### Activity Logs (Test Cases 36-39)
- **Test Cases**: 4 tests
- **Focus**: Audit trail and accountability tracking
- **Coverage**: Complete audit trails, action filtering, user filtering, and detailed log information
